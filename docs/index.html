Custom Enemy Navigation

The enemy system is planned as a grid-based A* pathfinder. The monster should navigate corridors, avoid static obstacles, and recalculate its route when the player moves or creates sound.

  • Grid-based node architecture for the base map
  • Dynamic rerouting during chase behavior
  • Sound events that can update the monster target
  • Hiding spots that can interrupt or redirect pursuit
